Table of pending cases

The table provides a dynamic, detailed list of pending cases that can be sorted and/or filtered by current status of each case (needs grossing, needs embedding, etc), subspecialty (Breast, Derm, GI, GU, etc), and colour code that indicates the delay encountered up to the current time.


Table columns:

The columns of the table include, in the following order from left to right, the case number, the date and time it was accessioned, the sub-specialty of the case, the status (needs grossing, needs embedding of blocks, needs microtomy, etc), the main specimen code, the number of specimens, number of blocks, number of slides (once known), the time allotted in hours to reach its current status (see below), the time that has passed in hours since the accessioning until the current time, and finally the percentage of time that has passed relative to the time alloted.


Color Codes:

Each case (row) has a color background to quickly visualize its performance status. A well-recognized colour scheme is that of traffic lights, where green indicates that a target is being met (for example, a case is being processed well within the default time frame assigned to this case type), red indicates that a target is not being met (for example, a case has spent more hours in the Histology Laboratory than what is expected), and amber indicates that the case is approaching (70% to 100%) its default time frame.


Displaying Case Events in Details:

You can review the detailed AMP events that have occured for any particular case by clicking with the mouse anywhere on its row. This opens another window that displays all the specimens of the case, and all the AMP events for each specimen, block and slide in real-time. You must close this window to be able to return to the main screen.


Table Sorting:

The cases can be sorted on any column by clicking the column header of interest. The soringt order can be toggled between ascending and descending order by clicking more than once on the column header.


Table Filtering:

You can restrict the table contents to display only a specific subset of cases by using the 3 drop-down lists in the menubar (2 in the Grossing module). See the "Menu Bar Help Page" for further details.


Formulas:

Color Codes are calculated based on number of business days/hours that have passed since accession time to the current time (HrsPassed). This is compared to the total time assigned to complete all tasks up to the case current stage (HrsAlloted). The red color code implies a ratio of HrsPassed/HrsAlloted more than 100%, amber awhen ratio is 70-100%, otherwise, the code is Green.
